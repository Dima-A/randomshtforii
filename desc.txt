Данная программа демонстрирует работу с разделяемой памятью (POSIX Shared Memory) без механизма синхронизации работы.
Программа работает следующим образом:
•	Общая часть для всех демонстраций:
1)	Создается объект разделяемой памяти через shm_open()
2)	Объект отображается в адресное пространство через mmap()
3)	В отображенной области размещается структура shared_data_t, содержащая:
-	Мьютекс (используется не всегда)
-	Счетчик
-	PID последнего процесса
-	Текстовое сообщение
•	Этап 1: Демонстрация Race Condition (demonstrate_race_condition())
1)	Мьютекс не используется и не инициализируется
2)	5 дочерних процессов одновременно увеличивают счетчик (каждый по 3 раза)
3)	Из-за отсутствия синхронизации возникает состояние гонки:
-	Процессы читают текущее значение counter
-	Между чтением и записью другой процесс может изменить counter
-	В результате финальное значение меньше ожидаемого 15
•	Этап 2: Демонстрация pthread_mutex_trylock() (demonstrate_with_trylock())
1)	Мьютекс инициализируется с атрибутом PTHREAD_PROCESS_SHARED
2)	Дочерний процесс захватывает мьютекс и удерживает его 3 секунды
3)	Родитель пытается захватить мьютекс без блокировки через trylock()
4)	Родитель получает EBUSY (мьютекс занят) и продолжает работу


Программа работает следующим образом:
•	Общая схема работы:
1)	Создание объекта разделяемой памяти через shm_open() — создается POSIX объект с заданным именем
2)	Установка размера через ftruncate() — задается размер сегмента под структуру данных
3)	Отображение в память через mmap() — объект отображается в адресное пространство процесса
4)	Инициализация мьютекса через pthread_mutex_init() с атрибутом PTHREAD_PROCESS_SHARED
5)	Размещение структуры shared_data_t, содержащей:
-	mutex — мьютекс для синхронизации (process-shared)
-	counter — счетчик (защищаемый ресурс)
-	last_pid — PID последнего изменявшего процесса
-	message — текстовое сообщение
6)	Порождение процессов через fork() для создания конкурентного доступа
7)	Синхронизация доступа с помощью pthread_mutex_lock() / pthread_mutex_unlock()
8)	Очистка ресурсов — уничтожение мьютекса, отмена отображения, удаление объекта памяти
•	Этап: Корректная синхронизация с мьютексом (demonstrate_with_mutex())
1)	Инициализация синхронизации:
-	Создается мьютекс с атрибутом PTHREAD_PROCESS_SHARED — критически важно для работы между разными процессами
-	Мьютекс инициализируется в разделяемой памяти, доступной всем процессам
2)	Операции с мьютексом:
pthread_mutex_lock() — блокирующий захват мьютекса:
-	Если мьютекс свободен → процесс входит в критическую секцию
-	Если мьютекс занят → процесс блокируется до освобождения
pthread_mutex_unlock() — освобождение мьютекса:
-	Делает мьютекс доступным для других процессов
-	Разблокирует ожидающий процесс (если есть)
3)	Действие программы:
Запускается 5 дочерних процессов
Каждый процесс 3 раза выполняет критическую секцию:
-	Выполняет pthread_mutex_lock() (захват мьютекса)
-	Читает текущее значение counter в локальную переменную
-	Имитирует длительную операцию внутри секции (usleep)
-	Увеличивает counter на 1 (current + 1)
-	Обновляет last_pid текущим PID
-	Обновляет текстовое сообщение
-	Выводит информацию об обновлении
-	Дополнительная задержка внутри секции (usleep)
-	Выполняет pthread_mutex_unlock() (освобождение мьютекса)
-	Кратковременная задержка вне критической секции
